# v0.5 Release Prep Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Prepare and publish a review branch for the dated `v0.5.20260410` release without touching `master` until explicit approval.

**Architecture:** Work in a clean release worktree based on `origin/develop`, make only release metadata and documentation changes, verify locally, and push `release/0.5.20260410` for review. The `master` fast-forward and GitHub release publication are deliberately separated into an approval-gated follow-up.

**Tech Stack:** Git, Maven, Java 21, Markdown, existing GitHub Actions release workflow.

  broader singleton lifecycle testing.

- S3K is not yet a full-game path. Non-AIZ/HCZ zones still need object, event, scroll, boss, and PLC
  parity work.
- Bonus stages are active implementations, not pixel-perfect finished systems.
- The level editor foundation exists, but the editor remains a future release goal.

### Assessment

The shared-system hardening goal was exceeded. The S3K expansion goal moved from AIZ-only depth into
early HCZ breadth, but full S3K zone coverage remains a v0.6+ theme.
