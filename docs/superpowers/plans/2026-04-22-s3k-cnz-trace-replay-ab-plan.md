# S3K CNZ Trace Replay - Workstreams A + B Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a "discard-on-title-reset" profile to the S3K BizHawk trace recorder and use it to produce a CNZ1+CNZ2 trace fixture + `TestS3kCnzTraceReplay` test scaffold, ending with a baseline divergence report that drives future workstreams C-G.

**Architecture:** Additive edits to `tools/bizhawk/s3k_trace_recorder.lua` that introduce a new profile (no behavioural change to existing profiles). The recorder gains a title-screen-detected reset hook that drops in-progress recording state and latches onto the next level. Test scaffold mirrors `TestS3kAizTraceReplay`.

**Tech Stack:** Lua 5.4 (BizHawk 2.11 Lua console), Genesis Plus GX core, Java 21 + JUnit 5 (trace replay harness), Maven, PowerShell (record_s3k_trace.bat), optional Python for BK2/CSV inspection.

## Busy-Worktree Caveat (READ FIRST)

This plan executes inside `.worktrees/feature/ai-s2-s3k-trace-recorder-v3`, which currently has **two other agents actively editing code**: one working on S3K AIZ traces (touching `Sonic3kAIZEvents.java`, `s3k_trace_recorder.lua`, `AbstractTraceReplayTest.java`, `TraceReplayBootstrap.java`, `HeadlessTestFixture.java`, `HeadlessTestRunner.java`, `TestS3kAizReplayBootstrap.java`) and one on S2 EHZ1 traces (touching S2-side code). Rules of engagement for any agent executing a task in this plan:

1. **Before every task: `git status` and `git log --oneline -5`.** Know which files are dirty and which commits have landed since the last task.
2. **All edits to the recorder must be additive.** Never delete or alter an existing profile's behaviour. Always add a new branch / new profile.
3. **Never modify files that are only touched in C-G** except as directed. This plan is A+B only. No pre-emptive boss/cutscene/object edits.
4. **Shared files at risk of conflict:** `tools/bizhawk/s3k_trace_recorder.lua`, `tools/bizhawk/record_s3k_trace.bat`, `src/test/java/com/openggf/tests/trace/AbstractTraceReplayTest.java`, `src/test/java/com/openggf/tests/HeadlessTestFixture.java`, `src/test/java/com/openggf/tests/HeadlessTestRunner.java`, `src/test/java/com/openggf/tests/trace/TraceReplayBootstrap.java`.
5. **If another agent has already added the same profile or the same helper, reuse it.** Check the current file contents before writing a fresh edit.
6. **One commit per task.** Never batch multiple tasks into one commit - the other agents may rebase or cherry-pick around you.
7. **Commit trailers: the repo's `prepare-commit-msg` hook appends the required block; fill in `updated` / `n/a` per the CLAUDE.md trailer/file mapping rule.**
8. **If a merge conflict materialises** mid-task, stop, run `git stash` to park your work, investigate on `develop` what the other agent committed, then reapply - do not silently resolve conflicts in shared files.
9. **BizHawk recording is stateful.** Running the recorder overwrites `tools/bizhawk/trace_output/`. Do not run recordings while another agent is recording.

If any of the tasks below refer to a helper or constant that the shared file already defines (because the AIZ agent landed it first), **skip the creation step and only wire it up**.

**Files:**

  ```lua
  local function reset_recording_state()
      -- Abandon any in-progress recording buffers and reset all frame
      -- counters so the next level-gameplay entry starts fresh.
      if physics_file then physics_file:close() end
      if aux_file then aux_file:close() end
      physics_file = nil
      aux_file = nil
      started = false
      trace_frame = 0
      bk2_frame_offset = 0
      start_x = 0
      start_y = 0
      start_zone_id = 0
      start_zone_name = "unknown"
      start_act = 0
      prev_status = 0
      prev_routine = 0
      prev_ctrl_lock = 0
      prev_player_mode = nil
      known_objects = {}
      emitted_checkpoints = {}
      last_zone_act_state_key = nil
      os.remove(OUTPUT_DIR .. "physics.csv")
      os.remove(OUTPUT_DIR .. "aux_state.jsonl")
      os.remove(OUTPUT_DIR .. "metadata.json")
  end
  ```

- [ ] **Step 3:** Add a reset-detection helper near `should_start_recording`.

  Insert after `is_level_gated_reset_aware_profile`:
  ```lua
  local function should_discard_and_reset(game_mode)
      if not is_level_gated_reset_aware_profile() then
          return false
      end
      if not started then
          return false
      end
      -- The only transitions that legitimately pull the player out of
      -- Game_Mode 0x0C mid-recording without a normal level-completion
      -- path are (a) pause + A back to title, or (b) soft power-on
      -- loop. Both surface as Game_Mode in {SEGA, TITLE, LEVEL_SEL}.
      return game_mode == GAMEMODE_SEGA
          or game_mode == GAMEMODE_TITLE
          or game_mode == GAMEMODE_LEVEL_SEL
  end
  ```

- [ ] **Step 4:** Wire the reset-check into `on_frame_end`.

  In `on_frame_end`, directly after the line `local game_mode = mainmemory.read_u8(ADDR_GAME_MODE)` and before `if not started then`:

  ```lua
      if should_discard_and_reset(game_mode) then
          print(string.format(
              "level_gated_reset_aware: detected soft-reset (Game_Mode=0x%02X) at trace frame %d. Discarding recording and re-arming.",
              game_mode, trace_frame))
          reset_recording_state()
          -- Fall through so the same frame can immediately re-arm if
          -- the player is somehow back in gameplay (shouldn't happen
          -- but is harmless).
      end
  ```

- [ ] **Step 5:** Update the finalisation branch so the new profile does not finalise just because gameplay ended (it should only finalise at BK2 end).
